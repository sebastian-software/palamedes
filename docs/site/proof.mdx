# Confidence Narrative

Palamedes should be presented as a tool that earns confidence visibly: the
message is positive, but the repo still shows the work behind it.

## What We Can Safely Claim

- the stack stays consistent across verified integrations for Next.js, TanStack
  Start, SolidStart, Waku, and React Router
- the runtime model is stable around `getI18n()`
- message identity is stable around `message + context`
- macro transform, extraction, catalog update, audit, and compile paths are
  backed by native tooling
- the example matrix is exercised in CI with browser interaction checks

## What The Confidence Stack Looks Like

### 1. The example matrix

The strongest repo-level evidence is the framework matrix:

- five framework families
- cookie and route locale strategies
- SSR-visible output
- localized server actions or server functions
- browser-driven interaction verification

This is the clearest evidence that Palamedes is broader than one good adapter.

### 2. Versioned screenshots

The screenshot set turns CI verification into something people can see. It
makes the cross-framework claim easier to communicate without relying on hosted
demos.

### 3. Benchmarks

Benchmarks matter, but only in the scope the repo can honestly support:

- transform
- extract
- catalog update
- catalog artifact compile

The benchmark language should stay disciplined:

- no universal performance claims
- no "Rust means fast" shortcut
- no benchmark charts without methodology

### 4. Architectural evidence

The design docs and ADRs matter because they explain why the product is credible:

- Rust core
- host-neutral compiled artifacts
- catalog semantics delegated to `ferrocat`
- small framework adapters

## Approved Confidence Language

Use language like:

- "verified across five framework families"
- "one runtime model across modern frameworks"
- "verified, not demo-only"
- "reproducible local benchmark commands"
- "browser-verified example matrix"

Avoid language like:

- "works everywhere"
- "best-in-class"
- "fastest"
- "universal results"
