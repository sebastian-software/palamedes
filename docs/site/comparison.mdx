# Comparison Messaging

> Internal messaging note. The published site lives in `site/`; this document
> records positioning language and safe comparison lines for maintainers.

Palamedes should be framed by product shape and daily developer experience, not
by a shallow feature checklist.

## Approved Comparison Line

Palamedes is for teams that like compile-time authoring and want the i18n stack
under it to feel smaller, steadier, and easier to trust.

## Lingui

Lingui is the closest direct comparison.

Use this framing:

- familiar authoring feel
- steadier runtime and identity model in Palamedes
- smaller adapter boundary
- narrower, more opinionated product shape

Do not frame Lingui as obsolete. Frame Palamedes as a sharper default for teams
already aligned with the same core instinct.

## next-intl

`next-intl` should be positioned as a stronger fit for teams that want a
Next.js-native, message-file-first center of gravity.

Use this framing:

- different default mental model
- some convergence on compile-time concerns
- not a clean benchmark peer to Palamedes end-to-end

## General Translation

General Translation should be framed as broader in product scope.

Use this framing:

- broader than an i18n runtime/tooling library
- includes wider translation workflow and service-oriented concerns
- useful adjacent signal, but a different category boundary

## Safe Repetition Lines

- "Palamedes is narrower than some alternatives on purpose."
- "The win is speed in service of a translation model that stays steady."
- "The same i18n habits keep working across framework boundaries."
- "The repo shows the work instead of asking for trust."
- "Palamedes keeps ICU semantics intact across the source, catalog, compile, and runtime stages it controls."

## ICU Claim Boundary

Do not say that every TMS preserves ICU or that Palamedes controls a TMS
round-trip. Public comparison copy should distinguish:

- the checked Palamedes pipeline proof;
- dated observations from official competitor and TMS documentation;
- TMS import/export behavior, which varies by product, format, and project
  configuration.

The approved proof and current source snapshot live in
`docs/icu-semantics-proof.md`.
