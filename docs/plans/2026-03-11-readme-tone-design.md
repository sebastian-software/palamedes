# README Tone Design

Date: 2026-03-11

## Goal

Make the public entry-point READMEs feel bolder and more memorable without making the infrastructure packages sound inflated or unserious.

## Scope

- Root README
- `@palamedes/vite-plugin`
- `@palamedes/next-plugin`
- `palamedes`

## Tone Decisions

- Use stronger language in the opening lines and value framing.
- Keep claims directional and plausible, not hype-only.
- Avoid vague comparative language like `faster`.
- Prefer more distinctive wording like `brutally fast`, `fast path`, and `built for modern apps`.
- Keep lower-level and highly technical package READMEs more restrained.

## Writing Pattern

- Headline/value proposition: bold
- Supporting paragraph: concrete
- Body sections: clear, helpful, and technically grounded

## Non-Goals

- No API or installation changes
- No repo-wide tone rewrite for the lower-level packages in this pass
